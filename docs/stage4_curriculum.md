## 二、为什么不直接按 IFD 升序排列？

这是面试中最容易被追问的点。

### 问题

如果直接按 IFD 从小到大排列：
```
[IFD=0.144] "写一个 Python 函数..."   (代码)
[IFD=0.189] "2018年7月有哪些会议..." (知识问答)
[IFD=0.192] "阐述扶贫攻坚的目标..."   (文本生成)
...  ↓ 全部简单任务扎堆在前半段
[IFD=1.133] "分析论坛内容..."         (推理)
[IFD=1.169] "写一个汽车行业Python..." (代码)
[IFD=1.216] "Python代码提取产品属性..." (代码)
```

**灾难性遗忘**：模型在前 200 步只看到简单的 QA 和文本生成，后 200 步突然全是高难度代码。学了后面的忘了前面的——因为不同难度带的任务类型天然分布不均。

### 结论

纯 IFD 排序 = 隐式地按 task_type 排序。课程学习不应该改变数据分布，只应该改变**曝光顺序**。

## 三、策略A：分带混洗 (Band-Shuffle)

### 一句话

按 IFD 分 N 个难度带，带内随机混洗打破类型聚集，带间递增保持"从易到难"的课程学习本质。

### 算法

```
1. 自动确定带边界（等分位数，如 p33, p67）
2. 每条数据按 IFD 落入对应 band
3. 每 band 内随机 shuffle
4. 按 band1 → band2 → band3 输出
```

### 参数

| 参数 | 值 | 理由 |
|------|-----|------|
| band 数 | 3 | 太少(2)→梯度不够细，太多(5+)→带间差异小 |
| 边界 | 自动等分位数[p33, p67] | 不预设具体值，适配数据分布 |
| 每带分配 | exactly equal | 163条/带，保证每个阶段有足够数据 |

### 结果

| Band | IFD范围 | 条数 | 主要类型 |
|------|---------|------|----------|
| 1 (简单) | 0.14-0.43 | 163 | 文本生成 47, 知识问答 45, 多轮对话 26 |
| 2 (中) | 0.43-0.85 | 163 | 推理分析 43, 多轮对话 30 |
| 3 (难) | 0.85-1.22 | 163 | 代码编写 70 |

### 面试话术

> "分带混洗的核心思想是——难度递增但类型混合。同带内随机打乱防止模型记住'越往后越难'的规律，带间递增保留课程学习收益。这比纯 IFD 排序好，比纯随机好。"

## 四、策略B：β退火加权采样 (Beta-Annealing)

### 一句话

不用硬分带。给每条数据一个**采样概率**（IFD 越低权重越高），β 逐步从 1.0 衰减到 0.1——早期倾向简单样本，后期接近均匀采样。

### 数学原理

**采样权重公式**：
```
P(样本 i) ∝ exp(-IFD_i * β / τ)
```

| 符号 | 含义 | 作用 |
|------|------|------|
| IFD_i | 样本 i 的难度 | 越高 → 被选概率越低 |
| β | 退火系数 | 大 → 简单样本权重指数级优势；小 → 接近均匀 |
| τ | 温度 | 控制分布锐度，τ=0.5 适中 |

**β 退火序列**：
```
β = [1.0, 0.775, 0.55, 0.325, 0.1]   (5步线性衰减)
```

### 直觉

| β 值 | 简单/困难权重比 | 有效样本占比 | 熵 | 训练阶段 |
|------|----------------|-------------|-----|----------|
| 1.0 | 38.9x | 57.1% | 5.88 | 早期：集中在最简单 57% |
| 0.55 | 7.5x | 81.8% | 6.08 | 中期：逐步引入更多数据 |
| 0.1 | 1.4x | 100% | 6.19 | 后期：接近均匀采样 |

关键观察：**β 从 1.0→0.1，权重比从 38.9x 降到 1.4x，熵从 5.88 增到 6.19**。这验证了退火过程确实让采样分布从"集中"逐步变为"均匀"。

### 为什么用 exp 而非线性权重？

IFD 是非对称分布的（左偏，少数样本 IFD>1.0）。exponential 能自适应地拉开简单/困难样本的权重差——β 大时简单样本获得指数级优势，β 小时自动平滑。线性权重做不到这种自适应。

### 面试话术

> "β退火的本质是让数据难度调度连续化。不需要预设分带边界——每条数据都有一个采样概率，β 大时简单样本概率高，β 逐步衰减到 0.1 后概率接近均匀。这样模型全程都看到全分布，只是在不同阶段关注不同难度。"

## 五、两种策略对比

| 维度 | 策略A (分带混洗) | 策略B (β退火) |
|------|-----------------|--------------|
| 难度过渡 | 硬切换（带边界突变） | 软过渡（β连续衰减） |
| 采样方式 | 同带均匀 | 全文按权重采样 |
| 灾难遗忘 | 带间切换有震荡风险 | 全程均匀混合，风险更低 |
| 自适应 | 固定边界，需要预调 | β 自动适配 |
| 适用规模 | 任意 | ≥500条效果更好 |
| 业界定位 | 经典 baseline | 2024 主流做法 |
| 面试价值 | 解释"为什么不能纯IFD排序" | 对比 β 退火曲线 vs 分带，更有深度 |

### 实践中如何使用？

训练时有两个选择：
1. **直接用 Band 序列**：epoch 1 = Band1×3 → epoch 2 = Band1+Band2 → epoch 3 = all
2. **用 β 权重采样**：每个训练步按 `sampling_weight` 从全集中采样 batch

阶段6 时我们会实际验证两种策略的下游训练效果差异。

## 七、面试速记卡

| 问题 | 回答要点 |
|------|----------|
| 为什么不纯IFD排序？ | task_type 与 IFD 相关 → 纯排序 = 隐式按类型排序 → 灾难性遗忘 |
| 分带混洗的核心思想？ | 带内随机打破类型聚集，带间递增保持课程学习收益 |
| β退火的本质？ | β从1.0衰减→0.1，采样从"集中简单"平滑过渡到"均匀全体" |
| 为什么用 exp 权重？ | IFD 非对称分布，exp 自适应拉开权重差 |
| β退火比分带好在哪？ | 软过渡避免"突然换档"震荡，不需要预设带边界 |
| 两种策略谁好？ | 看数据规模。<5000条→带分带足够；>5000条→β退火避免带内冗余 |
