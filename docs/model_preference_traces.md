# 合成模型的偏好痕迹：不同 API/模型合成指令数据的分布差异分析

> 这是第二层对比实验中产生的一个独立观察，值得单独记录为知识资产。

---

## 一、现象的发现

在 DeepSeek vs Qwen vs 同模型不同版本的多轮对比中，一个系统性规律浮现：

**同一套种子文本、同一套 prompt 模板、同一套 task_type 预分配策略——不同模型产出的指令数据在分布上存在显著且一致的结构性差异。**

不是在"质量好坏"，而是在"偏好痕迹"——每个合成模型都在它产出的数据上留下了可辨认的"指纹"。

### 实验证据

| 对比维度 | DS deepseek-chat | Qwen3.6-flash | Qwen3.6-plus | DS v4-flash(MaaS) | DS v4-pro(MaaS) |
|-----------|-----------------|---------------|--------------|-------------------|-----------------|
| task_type 分布差距 vs 基准 | 基准 | 1.09 | 1.18 | 1.49 | — |
| 推理分析占比 | 21% | 74% | 78% | 96% | ~18% |
| 代码编写产出 | 83 条 | 0 | 0 | 0 | ~20 条 |
| 平均输出长度 | 518 字 | 264 字 | 201 字 | 162 字 | 434 字 |
| JSON 格式遵从 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 是否遵循 task_type 随机指令 | ✅ | ❌ (全倒向推理) | ❌ (全倒向推理) | ❌ | ✅ |

### 同模型不同版本的差异

| | DS v4-flash (MaaS) | DS v4-flash (DS API) | DS v4-pro (MaaS) |
|---|---|---|---|
| 输出长度 | 409 字 | 518 字 | 434 字 |
| 格式遵从 | ❌ (审核拦截) | ✅ | ✅ |

| 速度 | ~8s/次 | ~1.5s/次 (受限频后变慢) | ~8s/次 |

## 二、这意味着什么

### 2.1 合成模型的选择在塑造训练数据的分布

不是抽象地说"不同模型产出不同数据"——而是在具体维度上存在可量化、可预测的偏斜。如果只用一家 API 合成 5000 条指令，你的 SFT 模型会带上这个 API 模型的"口音"：

- 更倾向于某类任务（如全部偏向推理分析）
- 更倾向于某种回答长度（264 字 vs 518 字差异近一倍）
- 更倾向于某种回答结构（Qwen 的回答偏短、偏叙述性；DS 的回答偏长、偏结构化）

### 2.2 这与 Model Collapse 的联系

2024-2025 年合成数据研究的核心议题之一是 **model collapse（模型坍塌）**——如果用同一个模型既合成数据又训练，迭代几轮后模型输出会坍缩为一个固定模式。

我们的实验虽然只做了一轮合成，但已经观察到"单模型偏好"的第一手证据：
- Qwen 系列（flash/plus）不遵循 task_type 随机指令，70-80% 全倒向推理
- 这意味着如果只用 Qwen 合成 5000 条 → 训练 → 模型会严重偏向推理任务 → 代码/对话能力退化

"多模型合成"不是花活——是防止模型坍塌的工程刚需。

### 2.3 API 端点对模型行为的影响

同一个 `deepseek-v4-flash`，通过 MaaS 调用和通过 DS 官方 API 调用，行为不同：
- MaaS 版：有内容审核拦截，代码类全杀，速度慢 50 倍
- DS API 版：无拦截，格式遵从正常，速度快 50 倍

这不是模型版本差异——MaaS 在中间加了安全网关，改变了 prompt 的到达形态。

## 三、工程启示

| 启示 | 具体做法 |
|------|----------|
| **永远对比 2+ 个合成源** | 不信任单一 API/model 的分布代表了"真实多样" |
| **切换 API 前做行为 POC** | 同模型名 ≠ 同行为。换 endpoint 时必须 1 种子 × 5 类型验证 |
| **混合同源数据是免费的鲁棒性** | v4-flash + v4-pro 混合，不是缺陷而是优势 |
| **task_type 预分配比随机指定可靠** | 代码里控制分布比信任模型服从 prompt 指令更可靠 |
| **记录合成模型的"指纹"** | 每次合成的模型+版本+endpoint 写入数据元信息，便于追溯 |

## 四、进一步的研究方向

1. **多模型混合策略的定量实验**：DS:Qwen:GLM 按 1:1:1 → 训练 → 评测 → 对比单一模型合成的基准
2. **Model Collapse 在小规模下的可复现性**：用 104M MiniMind 做 3 轮迭代合成→训练→合成，观测 loss 和输出分布的坍缩
3. **偏好痕迹的量化指标**：设计一个"模型指纹分数"——基于 n-gram 分布、句式复杂度、词汇多样性等维度，自动检测合成数据的模型来源
