## 1.1 N-gram 训练/评估数据分离

### 当前问题

阶段1和阶段3的 N-gram PPL 模型都是用**待评估数据自身**训练的：

```python
# 当前做法（有问题）
texts = [d["text"] for d in data]  # 待清洗/待评估的数据
ngram_model.train(texts)           # 用这批数据训练
ngram_model.perplexity(text)       # 再对同一批数据打分
```

**后果**：N-gram 模型记住了这批数据的统计分布 → PPL 值整体偏低（median=5.2）→ 区分力不足。

### 为什么这是错的

N-gram 模型的本质是：
```
PPL(text) = exp(-1/N × Σ log P(w_i | context_i))
```

如果模型是用待评估数据训练的，`P(w_i | context_i)` 反映的是"这个 token 在这批数据里的出现概率"——而脏数据的 token 分布也在这批数据里，所以脏文本的 PPL 也不会很高。

**打个比方**：你用一群人的平均身高做基准来筛选"谁太高了"——基准本身就包含了那些人，所以谁也不会被筛掉。

### 正确做法

```
分离训练与评估数据：

训练数据（外部干净语料）           评估数据（待清洗数据）
        │                              │
  ├─ 中文维基百科                    ├─ C4 网页语料
  ├─ 新闻语料（人民日报）            ├─ 合成指令
  └─ 高质量书籍                      └─ ...任何待评数据
        │                              │
        ▼                              ▼
  N-gram 哨兵模型 ───────────→ PPL 打分
  （独立训练，不碰待评数据）
```

### 实现方案

**数据源选择**：中文维基百科 dump（`zhwiki-*`）

- ModelScope 有现成的 `wikitext-zh` 子集，~500万 tokens
- 或者直接从 `datasets` 加载 `wikipedia 20220301.zh`

**代码改动量**：`stage1_clean_pipeline.py` 和 `stage3_quality_eval.py`

```python
# 改后（正确做法）
# 1. 用外部干净语料训练哨兵模型（只需一次）
reference_texts = load_wiki_corpus()  # 中文维基
ngram_model.train(reference_texts)    # 独立训练
torch.save(ngram_model, "models/ngram_sentinel.pt")

# 2. 对任何待评数据，加载哨兵模型打分
ngram_model = torch.load("models/ngram_sentinel.pt")
ppl = ngram_model.perplexity(text)  # PPL反映的是"偏离正常中文"的程度
```

### 面试回答模板

> "第一版我用待评估数据自训练 N-gram 来快速验证流程。但很快意识到——这会让 PPL 偏低且区分力下降，因为模型记住了待评数据的分布。改进方案是用中文维基百科独立训练哨兵模型，然后对任何新数据打分。这样 PPL 反映的是'偏离标准中文的程度'，而非'偏离这批数据均值的程度'——后者的基准会随数据变化，不可靠。"

### 改进后要验证什么

- 分离前后 PPL 均值对比（预期分离后 PPL 整体上升 2-3x）
- 分离前后 PPL 标准差对比（预期分离后 std 更大，区分力更强）
- 分离后 PPL 分布是否接近正态（更容易设定阈值）

## 1.2 规则过滤白名单机制

### 当前问题

阶段1有五条硬规则：

| 规则 | 阈值 | 误杀场景 |
|------|------|----------|
| 最短长度 < 10 | `min_text_length=10` | 短法条："第一条 本法…"（9字） |
| 非中文比例 > 55% | `max_non_chinese_ratio=0.55` | 技术文档含大量英文专有名词 |
| 重复字符比 > 40% | `max_repeat_char_ratio=0.40` | 基本不会误杀 |
| URL > 20 | `max_url_count=20` | 基本不会误杀 |
| 有效字符比 < 50% | `min_effective_ratio=0.50` | 诗歌、古文（标点/空格多） |

### 白名单设计

白名单不是"直接保留"——而是在规则判定前先检查是否命中白名单模式，命中则跳过该条规则：

```python
WHITELIST_PATTERNS = {
    "length": [
        r"^第[一二三四五六七八九十百千]+[条章节]",  # 法律条文："第一条…"
        r"^[《「].+[》」]$",                     # 标题："《红楼梦》"
    ],
    "non_chinese": [
        r"^[A-Z][a-z]+[\s，、].*[一-鿿]",  # 英文专有名词+中文
        r"\b(API|CPU|GPU|HTTP|JSON|SQL|CSS|HTML)\b",       # 技术术语
    ],
    "effective_ratio": [
        r"^[一-鿿，、。]{3,}$",       # 纯中文短句
    ],
}
```

### 面试回答模板

> "规则过滤是快刀，但也有切错的时候。我加了三组白名单模式：法律条文（'第X条'开头）、英文技术术语+中文混排、纯中文短句。当文本命中白名单时，即使它不通过长度/非中文/有效字符规则，也会被保留。白名单不是降低标准——它只是说'这看起来像正常数据，给个机会让后面的 PPL 来判断'。"

### 改进后要验证什么

- 白名单救回了多少条被规则过滤的数据
- 这些被救回的数据 PPL 是否在合理范围（如果在 PPL 也没过，说明确实该弃）
- 误杀率从 X% 降到 Y%

## 1.3 人工抽样验证

### 当前问题

阶段1从 3679 条过滤到 3082 条，丢了 597 条。但我们从来没有逐条看过——这些被丢掉的 597 条**到底是不是垃圾**？

### 为什么面试必问

面试官听到"我过滤掉了 16% 的数据"时，第一个问题是："你确定过滤掉的都是垃圾吗？"

如果你回答"应该是"，他会追问"你看过吗？"——你卡住了。

如果你回答"我看过——从 597 条中随机抽了 50 条，46 条确认是垃圾、4 条是误杀，准确率 92%"——你赢了。

### 做法

从 `data/passed_rules.jsonl` 和 `data/cleaned.jsonl` 的差集中随机抽 50 条：
- 规则过滤拦截的（passed_rules 以外的）
- PPL 过滤拦截的（PPL > 30）
- LSH 去重拦截的（被判定为重复）

每类抽 15~20 条，逐条判断：
```
✅ 正确过滤 — 确实是垃圾
❌ 误杀 — 这条数据其实可以用
⚠️ 边界 — 不好判断，待定
```

### 面试回答模板

> "清洗管线从 3679 条中丢了 597 条。我从中随机抽了 50 条人工审查——其中 46 条确实是垃圾（正确率 92%），3 条是边界情况（质量不高但勉强可用），1 条是明显误杀（短法条被长度规则误杀——这就是我后续加白名单的原因）。所以管线的精度约 92%，召回方向偏保守——宁可多杀也不能漏垃圾进训练集。"

## 1.4 PPL 评测集建立

### 当前问题

阶段1和阶段3的 PPL 都用自训练模型——除了 1.1 讲的分离问题，还有一个问题：**没有固定评测集来校准 PPL 阈值**。

当前 PPL=30 这个值是"在这批数据上网格搜索出的最优"——换一批数据就得重新搜。

### 解决方案

建立一个**标准 PPL 评测集**——从干净语料 + 人工构造脏语料中取样：

```
标准评测集构成：
  ├─ 高质量中文 (50条)：维基百科、新闻社论 → 预期 PPL 低
  ├─ 一般中文 (50条)：论坛帖子、自媒体文章 → 预期 PPL 中
  └─ 劣质中文 (50条)：机器翻译垃圾、纯符号、乱码 → 预期 PPL 高
```

然后用这个评测集：
1. 训练 N-gram 哨兵模型 → 对评测集打分 → 得到三类文本的 PPL 分布
2. 设置阈值 = 高质量 PPL 上边界 × 1.5（防过严格）
3. 阈值不再随数据变化——成为可复用的标准

### 实现

```python
# utils/ppl_benchmark.py
class PPLBenchmark:
    def __init__(self):
        self.dataset = load_benchmark()  # 150条标注文本
    
    def calibrate_threshold(self, ngram_model):
        """基于标注数据自动推荐阈值"""
        quality_ppls = [ngram_model.perplexity(t) for t in self.high_quality]
        garbage_ppls = [ngram_model.perplexity(t) for t in self.low_quality]
        # 阈值 = 高质量 P95 × 1.5
        threshold = np.percentile(quality_ppls, 95) * 1.5
        return threshold
```

### 面试回答模板

> "PPL=30 这个值是当前数据上的网格搜索结果。为了让阈值可复用——我建了一个 150 条的 PPL 评测集，包含高质量中文、一般中文、劣质中文各 50 条。N-gram 哨兵模型训练后先在这个评测集上校准——阈值 = 高质量文本 P95 × 1.5。这样换任何领域数据，阈值都是基于这个固定评测集自动计算的，不需要重新网格搜索。"

## 第一层改进汇总

| # | 改进 | 核心原理 | 面试答案关键词 |
|---|------|----------|---------------|
| 1.1 | N-gram 分离 | 训练与评估独立，防止分布记忆 | "哨兵模型"、"偏离标准中文" |
| 1.2 | 白名单 | 规则误杀补偿，常见模式豁免 | "法律条文模式"、"技术术语豁免"、"给 PPL 判断的机会" |
| 1.3 | 抽样验证 | 精确量化过滤准确率，而非"应该" | "92%准确率"、"1条误杀是短法条" |
| 1.4 | PPL 评测集 | 阈值可复用性，标准校准 | "150条标注集"、"P95 × 1.5"、"自动校准" |

## 面试串联话术

> "第一轮跑通后，我对管线做了四项基础加固。第一，N-gram 哨兵模型改用中文维基百科独立训练，不再用待评估数据自训练——这样 PPL 反映的是'偏离标准中文的程度'而非'偏离自己分布的程度'。第二，规则过滤加了白名单——短法条、技术术语+中文混排不会被误杀。第三，我从被过滤的 597 条中抽查了 50 条——92%确实是垃圾，让我能精确说出准确率。第四，建了一个 150 条的标准 PPL 评测集，阈值可以自动校准，不再随数据变化而变化。"
