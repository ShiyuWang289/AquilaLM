# AquilaLM 知识框架

> 面试前最后一次复习用。每完成一个阶段更新。最后更新：2026-06-17。

## 一、项目总览：一句话定位

> **AquilaLM 是一个以数据闭环为核心驱动的轻量级大模型训练栈，聚焦于后训练阶段的数据工程与对齐，通过构建"数据清洗→指令合成→质量评估→课程学习→反馈闭环→端到端验证"六阶段自进化循环，用数据质量的持续提升带动模型能力的增长。前五阶段已完成并归档，阶段6（端到端训练验证）待 GPU 执行。**

**面试话术模板**：

> "这个项目的出发点是：模型能力的上限由数据质量决定。我不想只做调参侠，所以自己搭建了一套完整的数据飞轮系统——从原始网页语料开始，经过清洗、合成、评估、排序、到反馈闭环，六个阶段在本地 CPU/GPU + API 上全部跑通。最关键的发现是：Evol-Instruct 进化后的指令 IFD 难度比 Self-Instruct 高 31.3%，用下游客观指标量化验证了上游进化策略的有效性——这正是数据飞轮'反向驱动'逻辑的第一手证据。"

## 三、分模块详解

### 模块1：数据清洗流水线

**定位**：飞轮的入口过滤器。

**实际实现**（非调库，面试加分项）：

| 层级 | 方法 | 关键决策 | 结果 |
|------|------|----------|------|
| 规则过滤 | 5条正则链 | non_chinese 最活跃（C4网页特征） | 3679 → 3114 条 |
| N-gram PPL | **手写 trigram 回退模型** | 不用 KenLM，自己实现概率回退+smoothing | PPL=30 |
| MinHash+LSH | datasketch | 2×2网格搜索，最终 Jaccard=0.60 | 3114 → 3082 条 |

**健康度验证**：长度 std -70%，Bigram 多样性 +36%

**面试必问**：
- "为什么不用 KenLM？" → 手写 N-gram 能讲清底层回退机制和 PPL 公式推导，面试时更说服力
- "Jaccard 阈值怎么定？" → 不拍脑袋，2×2 网格搜索验证方向性影响
- "怎么证明清洗有效？" → 三套验证：分布健康度 + 抽样人工 + 下游训练 loss（阶段6补）

---

### 模块2：指令合成与偏好数据构造

**定位**：飞轮的数据生产车间。

**两级合成管线**：

```
Self-Instruct (v4flash, temp=0.8)
  → 80种子 × 5条/种子 → 379条有效 → PostFilter → 375条

Evol-Instruct (v4pro, temp=0.3)
  → 150候选，5种进化算子 → 130条成功 (87%)

合并 → 505条 → synthesized.jsonl
```

**任务类型分布**：代码 83 / 推理 80 / 文本生成 73 / 对话 72 / 知识问答 63（最悬殊 83 vs 63 < 25%，task_type 随机指定有效）

**DPO 偏好对**：40 对（代码 20 + 数学 20），总费用 ¥2.20

**关键踩坑**（面试讲故事用）：
- 代码执行判定成功率仅 5% → 引入 LLM-as-Judge (v4pro) 兜底
- max_tokens=10 被 v4pro 推理链吃光 → 改 500
- VPN 断开后 231 条全废 → 加断点续跑 + checkpoint

---

### 模块3：数据质量评估体系

**定位**：飞轮的质检中心。

**三维协同**：

| 维度 | 方法 | 模型 | 结果 | 作用 |
|------|------|------|------|------|
| PPL | 复用阶段1 trigram | N-gram | mean=3.9, 2.2-6.0 | 基础流畅门槛 |
| IFD | loss(answer\|instr)/loss(ans) | GPT-2 102M GPU | mean=0.639 | 难度分级器 |
| Diversity | 余弦相似度矩阵 | MiniLM-L12-v2 | 0.729 (1-mean_sim) | 全局监控 |

**核心发现** ⭐：SI 的 IFD=0.588，EI 的 IFD=0.773，差距 +31.3%。**用下游客观指标量化验证了 Evol-Instruct 的进化算子确实提升了指令复杂度。**

**面试必问**：
- "IFD 具体怎么算？" → 用 label masking（instruction 部分 label=-100），只在 answer token 上算 cross-entropy loss
- "为什么用 GPT-2 不用大模型？" → 1) IFD 只需相对排序 2) 模型越小越诚实（少记忆效应） 3) 102M 在 RTX 4060 上 20 秒跑完 505 条
- "三维能互相替代吗？" → 不能。PPL 测通顺、IFD 测因果、Diversity 测全局分布

---

### 模块4：课程学习排序器

**定位**：将 IFD 分数转化为可训练的难度序列。

**两套策略**（面试时对比着讲）：

| | 策略A：分带混洗 | 策略B：β退火加权采样 |
|---|---|---|
| 难度过渡 | 硬切换（3带边界） | 软过渡（β连续衰减） |
| 采样方式 | 同带均匀 | 全文按权重采样 |
| 业界定位 | 经典 baseline | 2024 主流 |
| 核心参数 | 等分位边界 [0.43, 0.85] | β:1.0→0.1, 权重比 39x→1.4x |

**为什么不直接按 IFD 升序排列？**
→ task_type 与 IFD 天然相关（知识问答偏简单，代码偏困难）。纯排序 = 隐式按类型排序 = 灾难性遗忘。分带混洗保住了"从易到难"的课程学习本质，又打破了类型聚集。

**面试追问**："β退火的数学原理？" → 采样权重 P ∝ exp(-IFD × β / τ)，β 大时简单样本指数级优势，β 逐步衰减到 0.1 后概率接近均匀。熵从 5.88 → 6.19 量化了从"集中"到"均匀"的过渡。

---

### 模块5：反馈闭环调度器

**定位**：整个飞轮的方向盘。让下游训练评估结果反向驱动数据策略参数。

**三大组件**：

```
评估结果 (8维信号)
  → SignalNormalizer: 原始指标 → 0~1 偏差分数
  → DecisionEngine: 13参数权重矩阵, Δparam = Σ(signal_deviation × weight)
  → ParameterAdjuster: 三种类型 (ratio/count/threshold), clamp ±30%, 自动备份
  → 更新 config.yaml → 触发重新合成
```

**权重矩阵设计原则**：每个参数受 2-3 个信号驱动，主要信号权重 ≥ 0.5，因果关系可解释。

**四种 mock 场景验证**（阶段6训练未做，先用模拟信号验证逻辑）：
- **drift**：多样性坍塌 + 对话崩 → ↑温度 + ↑多轮进化 + ↓去重阈值
- **hard**：代码/推理崩 → ↑代码采样 + ↑进化量
- **noise**：Loss 震荡 + 数据无效 → ↑种子PPL + ↑一致性阈值
- **baseline**：轻微偏差 → 几乎全 hold

**真实信号注入**：即使没有阶段6，也能从已有产物提取 ifd_gap=0.185 和 effective_ratio=0.33。

**面试必问**：
- "闭环是全自动的吗？" → "当前是半自动——信号归一化、权重决策、参数更新全部自动化，但策略调整的规则（权重矩阵）是人工设计的。这让我能深入理解每个参数的影响，为未来完全自动化积累规则。下一步升级路径是 SPIN 自博弈——模型自己生成数据、自己评判、自动决定训练策略。"
- "怎么防止过调？" → 单轮 clamp ±30%，自动备份支持 revert，多轮渐进收敛

## 五、全链路数据流

```
C4 3679条
  → [阶段1] 规则+PPL+LSH → cleaned.jsonl (3082条)
  → [阶段2] SeedCurator(80种子) → Self-Instruct + Evol-Instruct → synthesized.jsonl (505条)
  → [阶段3] PPL + IFD(GPT-2) + Embedding → scored.jsonl (505条)
  → [阶段4] PPL过滤(16条) → Band-Shuffle / β退火 → curriculum.jsonl (489条)
  → [阶段5] 评估信号归一化 → 权重矩阵决策 → 参数调整 → 回写 config.yaml
  → [阶段6] ⬜ GPU训练 + 下游评测 → 生成新一轮评估信号 → 回到阶段5
```
