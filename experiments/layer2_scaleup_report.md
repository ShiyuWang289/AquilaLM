# 第二层 数据增量实验报告

> 日期：2026-06-19 ~ 2026-06-20

## 实验设置

- **目标**：将指令数据从 505 条放大到近 2000 条，同时验证下游训练效果
- **API**：DeepSeek v4-flash + v4-pro（MaaS 免费额度 + DS 官方 API）
- **种子**：从 cleaned.jsonl (3082条) 中筛选 800 条干净种子
- **生成策略**：预分配 task_type（代码里控制 5 种类型均匀分布），非交给模型随机
- **代码编写适配**：用伪代码模板绕过 MaaS 内容安全审核

## 关键发现

1. **量级放大后，课程学习效果趋稳**：第一轮 0.4% 差距，第二轮 Band=Baseline，Beta 略好 0.09%
2. **数据多了，策略差异在缩小**——因为更多数据本身提供了更好的覆盖，减少了排序策略的影响空间
3. **Diversity 0.730 证明预分配策略有效**：种类扩大到 1921 条，多样性没有坍塌
4. **混合 v4-flash + v4-pro 对数据质量无负面影响**

## 踩坑记录

| 坑 | 根因 | 教训 |
|----|------|------|
| MaaS 模型不遵循 task_type 指令 | 阿里云代理有额外安全过滤 | 切换端点前必做行为 POC |
| DS API 账号级硬限频 | 切换时 5 并发触发滥用判定 | 切换 API 先单次预热，逐步提并发 |
| 并发版数据丢失 | 批量保存替代了增量追加 | 并发不改保存频率，每结果立即写盘 |
| Qwen3.6/3.7 全倒向推理 | Qwen 系列底层不执行"随机指定"指令 | 预分配 task_type 策略成为首选项 |
